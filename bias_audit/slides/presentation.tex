\section{Contexte}
\sectiontransition{1}{Contexte et problématique}{primarycolor}

\begin{frame}{Pourquoi auditer un modèle ?}
\begin{block}{Question centrale}
Un modèle qui prédit correctement en moyenne est-il forcément juste pour tous les groupes ?
\end{block}

\vspace{0.35cm}
\begin{columns}[T]
\column{0.48\textwidth}
\textbf{\textcolor{primarycolor}{Performance}}
\begin{itemize}
  \item Accuracy
  \item F1-score
  \item ROC-AUC
  \item Qualité moyenne du modèle
\end{itemize}

\column{0.48\textwidth}
\textbf{\textcolor{dangercolor}{Fairness}}
\begin{itemize}
  \item Erreurs par groupe
  \item Faux positifs différenciés
  \item TPR/FPR par attribut sensible
  \item Impact humain des décisions
\end{itemize}
\end{columns}
\end{frame}

\begin{frame}{Dataset COMPAS}
\begin{columns}[T]
\column{0.56\textwidth}
\begin{itemize}
  \item Source : ProPublica, fichier réel \texttt{compas-scores-two-years.csv}.
  \item Cible : \texttt{two\_year\_recid}, récidive observée à deux ans.
  \item Attributs audités : \texttt{race} et \texttt{sex}.
  \item Taille après nettoyage : \textbf{6172 observations}.
  \item Usage : audit pédagogique, pas décision judiciaire automatisée.
\end{itemize}

\column{0.38\textwidth}
\begin{alertblock}{Point éthique}
Le label judiciaire n'est pas une vérité neutre : il peut refléter des biais de mesure, d'observation ou de procédure.
\end{alertblock}
\end{columns}
\end{frame}

\section{Mitigation}
\sectiontransition{4}{Réduction des biais}{dangercolor}

\begin{frame}{Méthodes testées}
\begin{columns}[T]
\column{0.5\textwidth}
\begin{itemize}
  \item \textbf{Reweighting} : pondérer les exemples.
  \item \textbf{Resampling} : rééquilibrer les couples groupe/label.
  \item \textbf{Fairlearn} : entraîner sous contrainte de fairness.
\end{itemize}

\column{0.5\textwidth}
\begin{itemize}
  \item \textbf{Adversarial PyTorch} : cacher l'attribut sensible dans la représentation.
  \item \textbf{Seuils par groupe} : ajuster la décision après entraînement.
\end{itemize}
\end{columns}

\vspace{0.4cm}
\begin{alertblock}{Attention}
Une mitigation doit être jugée sur deux axes : baisse du biais et maintien d'une performance utile.
\end{alertblock}
\end{frame}

\section{Recommandations}
\sectiontransition{5}{Recommandations et limites}{secondarycolor}

\begin{frame}{Recommandations principales}
\begin{enumerate}
  \item Ne jamais conclure avec l'accuracy seule.
  \item Publier les métriques par groupe : TPR, FPR, selection rate, precision.
  \item Prioriser Equalized Odds pour les contextes où les erreurs ont un impact humain.
  \item Garder une supervision humaine et une procédure de contestation.
  \item Documenter les limites du dataset COMPAS et des seuils utilisés.
\end{enumerate}
\end{frame}

\begin{frame}{Limites}
\begin{columns}[T]
\column{0.48\textwidth}
\textbf{\textcolor{dangercolor}{Données}}
\begin{itemize}
  \item Labels judiciaires imparfaits
  \item Groupes minoritaires très petits
  \item Corrélations et proxys
\end{itemize}

\column{0.48\textwidth}
\textbf{\textcolor{warningcolor}{Méthodes}}
\begin{itemize}
  \item Fairness multiple et parfois incompatible
  \item Mitigation parfois coûteuse en performance
  \item Pas de preuve causale
\end{itemize}
\end{columns}
\end{frame}

\section{Conclusion}
\sectiontransition{6}{Conclusion}{primarycolor}

\begin{frame}{Message final}
\begin{exampleblock}{Conclusion}
Un modèle peut être statistiquement utile et socialement risqué. L'audit doit donc répondre à deux questions : \textbf{est-ce que le modèle prédit bien ?} et \textbf{pour qui se trompe-t-il le plus ?}
\end{exampleblock}
