\chapter*{Résumé}
\addcontentsline{toc}{chapter}{Résumé}
\markboth{Résumé}{Résumé}
\thispagestyle{fancy}

\begin{tcolorbox}[summarybox]
\large
Ce rapport explique, étape par étape, comment auditer les biais d'un modèle de classification sur le dataset réel \textbf{COMPAS} publié par ProPublica. L'objectif n'est pas seulement d'obtenir un modèle performant : il faut aussi comprendre si ses erreurs touchent certains groupes plus que d'autres.
\end{tcolorbox}

\vspace{0.8em}

\begin{tcolorbox}[successbox, title={Ce qui a été réalisé}]
\begin{itemize}
  \item Utilisation du dataset réel \texttt{compas-scores-two-years.csv}, avec \textbf{6172 lignes} après nettoyage.
  \item Étude des attributs sensibles \texttt{race} et \texttt{sex}.
  \item Entraînement de trois familles de modèles : régression logistique, forêt aléatoire et MLP.
  \item Comparaison des modèles avec et sans attributs sensibles dans les variables d'entrée.
  \item Mesure de la performance classique : accuracy, précision, rappel, F1-score et ROC-AUC.
  \item Mesure de la fairness : demographic parity, disparate impact, equalized odds, equal opportunity, TPR, FPR et selection rate.
  \item Tests de mitigation : reweighting, resampling, Fairlearn, seuils par groupe et adversarial debiasing avec PyTorch.
  \item Production de scripts, notebooks, tableaux, figures, rapport PDF et présentation PDF.
\end{itemize}
\end{tcolorbox}

\begin{tcolorbox}[infobox]
\textbf{Idée simple :} un modèle peut avoir une bonne note globale et rester injuste. Par exemple, si deux groupes obtiennent presque la même accuracy mais qu'un groupe reçoit beaucoup plus de faux positifs, le modèle peut créer une injustice concrète.
\end{tcolorbox}

\begin{tcolorbox}[warningbox, title={Conclusion en une phrase}]
Sur COMPAS, la performance globale est correcte, mais les écarts de fairness sur \texttt{race} sont importants : il ne faut donc pas déployer ce type de modèle sans audit, documentation, supervision humaine et suivi régulier.
\end{tcolorbox}

\chapter*{Introduction générale}
\addcontentsline{toc}{chapter}{Introduction générale}
\markboth{Introduction générale}{Introduction générale}

\begin{tcolorbox}[summarybox]
\centering\large\bfseries
La question centrale du projet :\\
un modèle qui prédit bien est-il forcément un modèle juste ?
\end{tcolorbox}

\section*{Pourquoi ce sujet est important}

Les modèles de classification sont utilisés pour prendre ou aider à prendre des décisions : accepter un crédit, détecter une fraude, recommander un traitement, classer un dossier, estimer un risque. Dans un exercice classique de machine learning, on regarde surtout si le modèle prédit correctement. On calcule alors une accuracy, un F1-score ou une ROC-AUC.

Mais dans un contexte humain sensible, cela ne suffit pas. Imaginons deux groupes de personnes. Si le modèle se trompe rarement sur le premier groupe mais souvent sur le second, la moyenne globale peut cacher un vrai problème. Le modèle peut sembler bon sur le papier tout en créant une différence de traitement.

\section*{Pourquoi COMPAS}

COMPAS est un outil utilisé aux États-Unis pour estimer le risque de récidive. Le dataset étudié ici vient de l'enquête de ProPublica sur les scores COMPAS. La cible \texttt{two\_year\_recid} indique si une personne a récidivé dans les deux ans. Les attributs sensibles étudiés sont principalement \texttt{race} et \texttt{sex}.

\begin{tcolorbox}[dangerbox, title={Point éthique majeur}]
Ce rapport ne dit pas qu'un algorithme doit décider seul dans un contexte judiciaire. Au contraire, il montre pourquoi ce serait dangereux. L'objectif est d'apprendre à auditer un modèle, pas à automatiser une décision de justice.
\end{tcolorbox}

\section*{Objectifs pédagogiques}

\begin{tcolorbox}[primarybox, title={Objectifs du projet}]
\begin{enumerate}
  \item Comprendre ce qu'est un biais algorithmique avec des mots simples.
  \item Préparer un dataset réel en gardant une trace claire des choix de nettoyage.
  \item Entraîner plusieurs modèles pour ne pas dépendre d'un seul résultat.
  \item Comparer performance globale et performance par groupe.
  \item Tester plusieurs méthodes de réduction de biais.
  \item Formuler des recommandations concrètes et prudentes.
\end{enumerate}
\end{tcolorbox}

\chapter{Comprendre le problème}

\begin{tcolorbox}[infobox]
Ce chapitre pose les bases. Il explique la classification, les erreurs d'un modèle et la différence entre performance et équité.
\end{tcolorbox}

\section{Qu'est-ce qu'un modèle de classification ?}

Un modèle de classification reçoit des informations en entrée et doit choisir une classe en sortie. Dans ce projet, le modèle reçoit par exemple l'âge, le nombre d'antécédents, le type d'accusation et d'autres informations disponibles dans COMPAS. Il doit prédire une classe binaire :

\begin{itemize}
  \item \textbf{0} : la personne n'est pas prédite comme récidiviste dans les deux ans ;
  \item \textbf{1} : la personne est prédite comme récidiviste dans les deux ans.
\end{itemize}

\begin{tcolorbox}[primarybox, title={Exemple très simple}]
Si un modèle prédit la météo, il peut classer demain comme \og pluie \fg{} ou \og pas de pluie \fg{}. Ici, au lieu de la météo, le modèle classe une personne selon un risque de récidive. Comme le sujet touche à des humains, une erreur peut avoir des conséquences beaucoup plus graves.
\end{tcolorbox}

\section{Les quatre types de résultats}

Pour comprendre les métriques, il faut comprendre la matrice de confusion. Elle compare la prédiction du modèle avec la réalité observée.

\begin{table}[H]
\centering
\caption{Lecture simple d'une matrice de confusion}
\begin{tabularx}{\textwidth}{>{\bfseries}l X}
\toprule
Terme & Explication simple \\
\midrule
Vrai positif (TP) & Le modèle prédit \og risque \fg{} et la personne récidive réellement. \\
Faux positif (FP) & Le modèle prédit \og risque \fg{} mais la personne ne récidive pas. \\
Vrai négatif (TN) & Le modèle prédit \og pas de risque \fg{} et la personne ne récidive pas. \\
Faux négatif (FN) & Le modèle prédit \og pas de risque \fg{} mais la personne récidive. \\
\bottomrule
\end{tabularx}
\end{table}

\begin{tcolorbox}[dangerbox, title={Pourquoi les faux positifs comptent beaucoup ici}]
Dans un contexte judiciaire, un faux positif peut signifier qu'une personne est considérée à risque alors qu'elle ne récidive pas. Si un groupe subit beaucoup plus de faux positifs qu'un autre, le modèle crée une différence de traitement.
\end{tcolorbox}

\section{Performance globale et fairness}

La performance globale répond à la question : \og Le modèle prédit-il souvent correctement ? \fg{} La fairness répond à une autre question : \og Le modèle se trompe-t-il de manière comparable pour les différents groupes ? \fg{}

\begin{table}[H]
\centering
\caption{Différence entre performance et fairness}
\begin{tabularx}{\textwidth}{>{\bfseries\color{primarydark}}l X X}
\toprule
Aspect & Question posée & Exemple de métrique \\
\midrule
Performance & Le modèle est-il bon en moyenne ? & Accuracy, F1-score, ROC-AUC \\
Fairness & Le modèle traite-t-il les groupes de manière comparable ? & DP, DI, EO, TPR, FPR \\
Interprétation & Peut-on faire confiance au résultat global ? & Analyse par groupe \\
\bottomrule
\end{tabularx}
\end{table}

\section{Nettoyage réalisé}

Le nettoyage vise à obtenir une base cohérente pour l'apprentissage automatique. Les lignes inutilisables sont retirées, les colonnes sont standardisées et les variables catégorielles sont préparées pour les modèles.

\begin{tcolorbox}[primarybox, title={Règle importante}]
Les attributs sensibles ne disparaissent jamais du projet : même si on les retire des variables d'entrée du modèle, on les conserve pour mesurer les écarts de traitement. Sans eux, il serait impossible de vérifier si le modèle est injuste.
\end{tcolorbox}

\section{Limites du dataset}

Un dataset judiciaire ne représente jamais parfaitement la réalité sociale. La variable \texttt{two\_year\_recid} dépend de ce qui a été observé, enregistré et poursuivi. Elle peut donc contenir des biais institutionnels ou historiques.

\begin{tcolorbox}[warningbox, title={À retenir}]
Le label \texttt{two\_year\_recid} n'est pas une vérité absolue. Il décrit une récidive observée dans les données. Si l'observation elle-même est biaisée, le modèle peut apprendre une partie de ce biais.
\end{tcolorbox}

\section{Découpage train, validation et test}

Le dataset est découpé en trois parties :

\begin{itemize}
  \item \textbf{Train} : sert à apprendre les paramètres du modèle.
  \item \textbf{Validation} : sert à comparer des réglages ou des seuils sans regarder le test final.
  \item \textbf{Test} : sert à mesurer les résultats finaux.
\end{itemize}

Quand c'est possible, le split est stratifié par une combinaison de la cible et des attributs sensibles. Cela évite qu'un groupe disparaisse accidentellement du test.

\section{Modèles comparés}

\begin{table}[H]
\centering
\caption{Modèles entraînés}
\begin{tabularx}{\textwidth}{>{\bfseries\color{primarydark}}l X X}
\toprule
Modèle & Idée intuitive & Pourquoi l'utiliser ? \\
\midrule
Régression logistique & Trace une frontière simple entre les classes. & Baseline claire, rapide et interprétable. \\
Random Forest & Combine plusieurs arbres de décision. & Capte des relations non linéaires et robuste aux variables mélangées. \\

\bottomrule
\end{tabularx}

\section{Deux politiques de features}

Les modèles sont entraînés dans deux situations :

\begin{enumerate}
  \item \textbf{\texttt{with\_sensitive}} : les variables sensibles peuvent être données au modèle.
  \item \textbf{\texttt{without\_sensitive}} : les variables sensibles sont retirées des features, mais gardées pour l'audit.
\end{enumerate}

\begin{tcolorbox}[warningbox, title={Erreur fréquente}]
Retirer \texttt{race} ou \texttt{sex} des features ne garantit pas l'absence de biais. D'autres variables, comme le lieu, le type d'accusation ou les antécédents, peuvent être corrélées aux attributs sensibles et servir de proxys indirects.
\end{tcolorbox}

\section{Répétitions multi-seeds}

Les expériences sont répétées avec les seeds \textbf{0, 1, 2, 3 et 4}. Une seed contrôle le hasard utilisé par certains algorithmes. Répéter l'expérience permet de vérifier si un résultat est stable ou s'il dépend d'un hasard favorable.

\section{Intervalles de confiance}

Le pipeline calcule aussi des intervalles de confiance bootstrap à 95\% sur des métriques clés. L'idée est simple : au lieu de croire qu'un seul score est parfait, on estime une zone plausible autour du score.

\chapter{Métriques utilisées}

\section{Métriques de performance}

\begin{table}[H]
\centering
\caption{Métriques classiques}
\begin{tabularx}{\textwidth}{>{\bfseries\color{primarydark}}l X X}
\toprule
Métrique & Question simple & Attention \\
\midrule
Accuracy & Quelle part des prédictions est correcte ? & Peut cacher les erreurs par groupe. \\
Précision & Quand le modèle dit \og risque \fg{}, a-t-il souvent raison ? & Important si les faux positifs sont graves. \\
Rappel / TPR & Parmi les vrais récidivistes, combien sont détectés ? & Important si les faux négatifs sont graves. \\
F1-score & Compromis entre précision et rappel. & Utile si les classes sont déséquilibrées. \\
ROC-AUC & Le modèle classe-t-il bien les risques du plus faible au plus fort ? & Ne suffit pas à juger la fairness. \\
\bottomrule
\end{tabularx}
\end{table}

\toprule
Métrique & Interprétation \\
\midrule

\begin{tcolorbox}[primarybox, title={Pourquoi Equalized Odds est centrale ici}]
Dans COMPAS, les erreurs n'ont pas toutes le même poids humain. Equalized odds est pertinente parce qu'elle regarde les faux positifs et les vrais positifs par groupe. Elle force à se demander : \og le modèle fait-il le même type d'erreurs selon les groupes ? \fg{}
\end{tcolorbox}

\section{Pourquoi atténuer les biais}

Une mitigation cherche à réduire les écarts entre groupes. Elle ne rend pas magiquement un système juste, mais elle permet de tester des compromis. Le but est de trouver une solution qui garde une performance utile tout en réduisant les disparités les plus problématiques.

\end{tcolorbox}

\chapter{Discussion critique}

\section{Pourquoi il n'existe pas une seule fairness}

La fairness est un ensemble de définitions parfois incompatibles. Un modèle ne peut pas toujours satisfaire toutes les métriques en même temps, surtout si les groupes ont des base rates différents. Il faut donc choisir la métrique en fonction du contexte.

\begin{tcolorbox}[infobox]
Dans un contexte judiciaire, Equalized Odds est plus pertinente que la simple parité démographique, car elle observe directement les erreurs. Ce qui compte n'est pas seulement le nombre de personnes classées positives, mais aussi qui est classé positif à tort.
\end{tcolorbox}

\section{Pourquoi l'accuracy peut tromper}

Une accuracy autour de 0.69 semble correcte. Pourtant, elle ne dit pas si les erreurs sont équilibrées entre groupes. Un modèle peut avoir la même accuracy pour deux groupes, mais faire plus de faux positifs dans l'un et plus de faux négatifs dans l'autre. L'impact humain n'est alors pas le même.

\section{Limites techniques}

\begin{itemize}
  \item Les groupes très petits produisent des métriques instables.
  \item Les résultats dépendent du split train/validation/test.
  \item Certaines méthodes de mitigation baissent fortement la performance.
  \item Les attributs sensibles peuvent être indirectement présents via des proxys.
  \item La causalité n'est pas démontrée : on observe des associations statistiques.
\end{itemize}

\section{Limites éthiques}

\begin{itemize}
  \item Un modèle ne doit pas remplacer une décision humaine dans un domaine judiciaire.
  \item Un dataset historique peut contenir des biais de police, de poursuite ou d'enregistrement.
  \item Réduire une métrique de biais ne garantit pas la justice réelle.
  \item Les personnes concernées devraient pouvoir comprendre, contester et demander une révision humaine.
\end{itemize}

\chapter{Recommandations}

\section{Recommandations techniques}

\begin{tcolorbox}[successbox, title={À mettre en place avant tout usage sensible}]
\begin{enumerate}
  \item Toujours publier les métriques globales et les métriques par groupe.
  \item Suivre TPR, FPR, selection rate, precision et F1-score pour chaque attribut sensible.
  \item Répéter les expériences avec plusieurs seeds.
  \item Documenter les choix de seuils et les compromis performance/fairness.
  \item Tester Fairlearn et adversarial debiasing, mais ne pas retenir une méthode seulement parce qu'elle diminue une métrique.
  \item Ajouter un monitoring si le modèle est réutilisé sur de nouvelles données.
\end{enumerate}
\end{tcolorbox}

\section{Recommandations de gouvernance}

\begin{tcolorbox}[warningbox, title={Gouvernance indispensable}]
\begin{itemize}
  \item Mettre en place une validation humaine pour toute décision sensible.
  \item Prévoir une procédure de contestation.
  \item Expliquer clairement les limites du modèle aux utilisateurs.
  \item Ne jamais présenter le score comme une vérité objective sur une personne.
  \item Réaliser un audit périodique, car les données peuvent changer avec le temps.
\end{itemize}
\end{tcolorbox}

\section{Recommandation finale}

La meilleure décision n'est pas de chercher \og le modèle parfait \fg{}. La meilleure décision est de mettre en place un système contrôlé : modèle documenté, métriques publiées, erreurs surveillées, seuils justifiés, supervision humaine et possibilité de recours.

% CHAPITRE 10
% ============================
\chapter{Conclusion}

Ce projet montre qu'un audit de biais doit aller beaucoup plus loin qu'un simple entraînement de modèle. Sur COMPAS, les modèles obtiennent une performance globale raisonnable, mais les écarts par groupe, surtout sur \texttt{race}, restent importants.

Le résultat principal est donc double :

\begin{enumerate}
  \item \textbf{Côté performance}, les modèles sont utilisables pour une étude technique, avec une accuracy autour de 0.69 et une ROC-AUC autour de 0.75.
  \item \textbf{Côté fairness}, les écarts observés interdisent toute conclusion naïve du type \og le modèle est bon donc il est juste \fg{}.
\end{enumerate}

\begin{tcolorbox}[summarybox]
\large
La leçon essentielle est simple : dans un domaine sensible, un modèle doit être évalué avec deux regards. Le premier regarde s'il prédit correctement. Le second regarde pour qui il se trompe, comment il se trompe, et quelles conséquences ces erreurs peuvent avoir.
\end{tcolorbox}
